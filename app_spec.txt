

# WorkflowMCP: Declarative Agent Orchestration - App Specification

## Overview
This specification defines the WorkflowMCP tool, an MCP server enabling declarative definition, execution, and monitoring of multi-agent workflows via YAML, integrated with Claude Code and supporting Copilot SDK extensions.  
It targets solo AI developers who need to quickly prototype and test agent-based systems without writing procedural glue.

## Problem Statement
Solo developers struggle to orchestrate multi-agent agents because they must write procedural code for each step, making workflows hard to visualize, modify, and share. Current methods involve ad-hoc scripts or complex orchestration tools that are overkill for solo use.

## Target Audience
Solo AI developers building MCP servers or agent-based tools who want to quickly define and test multi-agent workflows without boilerplate.

## Core Features
Each feature must be completable in roughly one build iteration.

### Feature 1: Declarative YAML Workflow Definition
**Description**: Define multi-agent workflows in YAML, specifying agents, tools, and step dependencies without writing imperative code.  
**Requirements**:
- Parse YAML schema and validate against workflow DSL.  
- Generate a directed acyclic graph (DAG) of agent invocations.  
- Provide hooks for static analysis and visualization.  
**Test Steps**:
1. `workflow-mcp parse --file example.yaml` -> Outputs JSON DAG representation.  
2. `workflow-mcp validate --file bad.yaml` -> Returns exit code 1 with error message.  
3. `workflow-mcp visualize --file example.yaml` -> Generates Mermaid diagram saved to `out.png`.

### Feature 2: RealвЂ‘time Agent Execution Monitoring
**Description**: Observe agent runs, collect metrics, and emit events for debugging and observability.  
**Requirements**:
- Subscribe to Claude Code execution stream via WebSocket.  
- Record start/end timestamps, token usage, and success/failure status.  
- Export metrics in Prometheus format and support logging to stdout.  
**Test Steps**:
1. `workflow-mcp monitor --workflow example` -> Streams JSON lines to stdout.  
2. `workflow-mcp monitor --workflow example --format prometheus` -> Exposes `:workflow_steps_total` metric.  
3. `workflow-mcp monitor --workflow example --test` -> Simulates a failure and verifies alert emission.

### Feature 3: Extensible SDK Integration (Copilot)
**Description**: Allow plugging in custom agent skills via the Copilot SDK, enabling languageвЂ‘modelвЂ‘powered tools.  
**Requirements**:
- Provide a Python package entrypoint `copilot_sdk.load_skill`.  
- Support dynamic loading of skill modules from local filesystem or git.  
- Expose a defined API for skill invocation (input: dict, output: dict).  
**Test Steps**:
1. `copilot_sdk load_skill --name summarizer` -> Loads module and returns handle.  
2. `copilot_sdk run_skill --handle h --input {\"text\":\"long article\"}` -> Returns summarized text.  
3. `copilot_sdk list_skills` -> Prints available skill names.

## Data Models
### Data Model: WorkflowModel
**Fields**:
- `name: str`  
- `steps: List[StepModel]`  

**StepModel** (nested):
- `agent: str`  
- `tool: str | None`  
- `depends_on: List[int] = []`

### Data Model: AgentModel
**Fields**:
- `name: str`  
- `skill_path: str`  
- `config: Dict[str, Any] = {}`

## Success Criteria
1. Running `workflow-mcp parse --file sample.yaml` yields a valid DAG JSON and exits with code 0.  
2. Running `workflow-mcp monitor --workflow test` produces Prometheus metrics endpoint at `http://localhost:9090/metrics`.  
3. Running `copilot_sdk list_skills` outputs at least three skill names: `summarizer, translator, planner`.  
4. Running `workflow-mcp validate --file invalid.yaml` returns non-zero exit code and prints `ERROR: Invalid step reference`.
